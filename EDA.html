O primeiro passo tomado foi analisar o desempenho das turmas nesses perodos para identificar discrepncias e enxergar padres.

Consideramos nossa primeira medida a mdia parcial. Ela nos d um overview do desempenho das turmas, pois considera todas a avaliaes excluindo a avaliao final.

Observando media_parcial vemos o perodo 2017.2 como aquele com o melhor desempenho mediano, onde, desde ento, o desempenho vem caindo, tendo o 2019.2 o menor dentre todos. Vamos destrinchar esse desempenho entre as principais atividades avaliativas: Provas, laboratrios e projeto.

Agrupamento

Between/totss

Outra medida comumente usada no kmeans comparar a distncia (quadrtica) entre o centro dos clusters e o centro dos dados com a distncia (quadrtica) entre os pontos todos nos dados e o centro dos dados.

Quebrando essa ideia para ficar mais fcil de entender:

Primeiro, o centro dos dados um ponto imaginrio na mdia de todas as variveis. um ponto que est no meio dos dados. Em uma situao onde cada ponto um grupo (e os grupos so os mais coesos possveis), a soma das distncias dos grupos para o centro dos dados igual soma da distncia dos pontos para o centro dos dados. Generalizando: se houver estrutura de grupos e ela estiver capturada pelo agrupamento, o somatrio da distncia do centro de cada grupo para o centro geral dos dados ser um valor alto.

Para medir para quais valores de k isso acontece, calculamos a distncia do centro de cada cluster para o centro dos dados e multiplicamos pelo nmero de pontos nesse cluster. Somando esse valor para todos os clusters, temos betweenss abaixo.

Se esse valor for prximo do somatrio total das distncias dos pontos para o centro dos dados (totss), os pontos esto prximos do centro de seu cluster. Essa proporo pode ser usada para definir um bom valor de k. Quando ela para de crescer, para de valer pena aumentar k.

O bom valor de k parece ser 3, que vai ser o nmero de clusters para o nosso algoritmo de agrupamento.
